# Requisitos de um Software

## Definição

Requisitos são especificações que definem funcionalidades e restrições de um projeto de software.

Eles servem como base para:
- Protótipo
- Design
- Implementação
- Testes

---

## Etapas da Análise de Requisitos

1. Levantamento  
2. Documentação  
3. Validação  
4. Gerenciamento  
5. Verificação e aprovação  

---

## Classificação dos Requisitos

Os requisitos são divididos em:
- **Requisitos Funcionais (RF)**
- **Requisitos Não Funcionais (RNF)**

---

## Requisitos Funcionais (RF)

Descrevem o que o sistema deve fazer.

Também podem ser classificados em:

- **Requisitos de Negócio**
- **Requisitos de Usuário**
- **Requisitos Técnicos**

### Importância
Os RFs promovem:
- Clareza
- Boa comunicação
- Melhor planejamento
- Direcionamento da equipe

### Exemplos

- Autenticação de usuário → *Requisito de usuário*  
- Gestão de inventário → *Requisito de negócio*  
- Persistência em banco de dados → *Requisito técnico*  

---

## Requisitos Não Funcionais (RNF)

Definem características, qualidades e restrições do sistema.  
Ou seja, descrevem **como o sistema se comporta**.

### Tipos

- **Requisitos de Produto**
- **Requisitos Organizacionais**
- **Requisitos Externos**

### Impacto

Os RNFs influenciam diretamente:

- Desempenho  
- Confiabilidade  
- Usabilidade  
- Segurança  
- Escalabilidade
