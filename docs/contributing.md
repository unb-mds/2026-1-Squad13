# Guia de Contribuição

## Organização de Branches

- `main`: versão estável
- `develop`: integração de desenvolvimento
- `feature/*`: novas funcionalidades
- `docs/*`: documentação

---

## Commits

Os commits devem possuir mensagens objetivas e padronizadas.

### Exemplos

- `docs: adiciona glossário`
- `feat: cria tela inicial`
- `fix: corrige validação`

---

## Pull Requests

Todo desenvolvimento deve ser realizado em branches separadas e integrado via Pull Request.

---

## Revisão

As alterações devem ser revisadas antes da integração na branch principal.
